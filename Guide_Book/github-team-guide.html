중요: 브랜치는 각자의 작업 공간입니다. 다른 사람의 브랜치에서는 작업하지 마세요!

🏗️ 4명 팀 브랜치 구조

main (최종 완성본) ├── develop (개발 통합 브랜치) ├── feature/frontend-팀원A (HTML 구조 작업) ├── feature/backend-팀원B (JavaScript 로직 작업) └── feature/design-팀원C (CSS 디자인 작업)

develop 팀장 개발 통합 관리 테스트 통합
feature/frontend-A 팀원A HTML 구조, 레이아웃 *.html, layout.css
feature/backend-B 팀원B JavaScript 로직 *.js, data.js
feature/design-C 팀원C CSS 디자인 style.css, theme.css

💻 VS Code에서 브랜치 생성

각자의 작업 공간 만들기

팀장 먼저: develop 브랜치를 만들어 팀원들에게 기본 구조를 제공하세요.

🔧 팀장이 develop 브랜치 생성

VS Code에서 하는 작업:
1단계: VS Code 하단 상태바에서 "main" 클릭
2단계: 드롭다운에서 "Create new branch" 선택
3단계: 브랜치명 "develop" 입력
4단계: Source Control 패널 (왼쪽 사이드바 3번째 아이콘) 열기
5단계: "Publish branch" 버튼 클릭하여 GitHub에 업로드

👥 팀원들이 개인 브랜치 생성

1
저장소 Clone: VS Code에서 Ctrl+Shift+P → "Git: Clone" → GitHub URL 입력
2
프로젝트 열기: Clone한 폴더를 VS Code에서 열기
3
브랜치 생성: 하단 "main" 클릭 → "Create new branch"
4
브랜치명 입력: "feature/frontend-본인이름" 형식으로 입력
5
GitHub에 업로드: "Publish branch" 클릭
// 브랜치명 예시 feature/frontend-김철수 feature/backend-이영희 feature/design-박민수

🔄 일일 협업 워크플로우

매일 이렇게 작업하세요

⚠️ 중요한 규칙: 작업 시작 전에는 반드시 Pull을 해서 최신 상태로 만드세요!

🌅 매일 작업 시작 전 (모든 팀원)

VS Code에서 하는 작업 순서:
1단계: Source Control 패널 열기 (사이드바 3번째 아이콘 🌿)
2단계: 상단의 "..." 메뉴 클릭 → "Pull" 선택
3단계: 하단에서 "develop" 브랜치로 이동
4단계: 다시 Pull 하여 develop 브랜치 최신 상태로 업데이트
5단계: 본인 브랜치로 이동 (예: feature/frontend-김철수)
6단계: "..." → "Merge Branch" → "develop" 선택

⚡ 작업 중 (각 팀원)

1
파일 수정: 본인 담당 파일들을 수정하고 저장
2
변경사항 확인: Source Control에서 수정된 파일들 확인
3
Stage 추가: 수정된 파일 옆의 "+" 버튼 클릭
4
커밋 메시지: 상단 입력창에 작업 내용 입력
5
커밋: "✓ Commit" 버튼 클릭
6
Push: "⬆️ Push" 버튼으로 GitHub에 업로드
// 좋은 커밋 메시지 예시 "로그인 페이지 HTML 구조 완성" "메인 페이지 CSS 스타일링 추가" "사용자 인증 JavaScript 로직 구현" "회원가입 폼 유효성 검사 추가"

🔄 작업 완료 후 (팀원 → 팀장)

Pull Request: 자신의 작업을 팀에 합치기 위한 요청입니다.
1
GitHub 웹사이트 접속
2
"Pull requests" 탭 클릭
3
"New pull request" 클릭
4
브랜치 선택: 본인브랜치 → develop 으로 설정
5
제목과 설명: 무엇을 작업했는지 자세히 작성
6
"Create pull request" 클릭
7
팀장에게 알림: 카톡으로 Pull Request 생성 알림

🎯 파일 충돌 방지 전략

같은 파일을 동시 수정할 때 문제 방지

충돌이란? 두 명 이상이 같은 파일의 같은 부분을 동시에 수정했을 때 발생하는 문제입니다.

📂 파일별 담당 분리

📁 프로젝트 루트 ├── 📄 index.html → 팀원A (프론트엔드) ├── 📄 login.html → 팀원A ├── 📄 board.html → 팀원A ├── 📁 css/ │ ├── main.css → 팀원C (디자인) │ ├── login.css → 팀원C │ └── responsive.css → 팀원C ├── 📁 js/ │ ├── app.js → 팀원B (백엔드) │ ├── auth.js → 팀원B │ └── data.js → 팀원B └── 📁 docs/ → 팀장 (문서)

🚨 공통 파일 수정 규칙

같은 파일을 여러 명이 수정해야 할 때:
1
사전 협의: 카톡/디스코드에서 미리 "OO.html 수정할게요" 공지
2
시간대 분리: 오전/오후 또는 요일별로 작업 시간 나누기
3
즉시 Push: 수정 완료하면 바로 커밋/푸시하여 다른 사람이 받을 수 있도록
4
작업 완료 알림: "OO.html 수정 완료했어요!" 단체톡에 알림

🚨 충돌 발생 시 해결법

문제 상황별 해결 방안

걱정하지 마세요! 충돌은 협업에서 자연스러운 현상입니다. VS Code가 해결을 도와줍니다.

⚡ VS Code에서 충돌 해결하기

충돌 발생했을 때 VS Code 화면:
1단계: 충돌이 발생한 파일에 "!" 표시가 나타남
2단계: 해당 파일을 열면 충돌 부분이 색깔로 구분되어 표시됨
3단계: 충돌 부분 위에 선택 버튼들이 나타남:
• "Accept Current Change" (내 코드 선택)
• "Accept Incoming Change" (상대방 코드 선택)
• "Accept Both Changes" (둘 다 포함)
• "Compare Changes" (비교해서 수동 선택)
4단계: 적절한 옵션 선택 후 파일 저장
5단계: Source Control에서 "Mark as Resolved" 클릭
6단계: 커밋 메시지에 "충돌 해결" 적고 커밋/푸시

🤝 팀 차원에서 충돌 해결하기

1
즉시 소통: "OO 파일에서 충돌 발생했어요" 단체톡에 알림
2
관련자 확인: 같은 파일 수정한 팀원과 직접 소통
3
코드 협의: 어떤 코드를 사용할지 함께 결정
4
해결 후 확인: 모든 기능이 정상 작동하는지 테스트
💡 팁: 충돌이 자주 발생한다면 파일을 더 세세하게 나누거나, 작업 시간을 더 명확히 분리하세요.

❓ 자주 묻는 질문 (FAQ)

문제 상황별 해결 방안

💡 팁: 문제가 발생하면 당황하지 말고 하나씩 차근차근 해결해보세요!

🤔 자주 발생하는 문제들

Q1: VS Code에서 GitHub 로그인이 안 돼요!
해결 방법:
  1. Ctrl+Shift+P → "GitHub: Sign in" 검색
  2. 브라우저가 자동으로 열리면 GitHub 로그인
  3. "Authorize Visual-Studio-Code" 클릭
  4. VS Code로 돌아와서 "Open" 클릭
Q2: 브랜치가 안 만들어져요!
해결 방법:
  1. 먼저 최소 한 번은 커밋이 있어야 합니다
  2. 빈 파일이라도 만들어서 첫 커밋을 해보세요
  3. Source Control에서 "Initialize Repository" 버튼 확인
  4. 그래도 안 되면 팀장에게 도움 요청
Q3: Push할 때 "permission denied" 오류가 나요!
해결 방법:
  1. GitHub 저장소에 초대받았는지 확인
  2. 초대 이메일에서 "Accept invitation" 클릭했는지 확인
  3. VS Code에서 올바른 GitHub 계정으로 로그인했는지 확인
  4. 팀장에게 Write 권한이 있는지 확인 요청
Q4: Pull Request는 어디서 만들어요?
해결 방법:
  1. 본인 브랜치에서 푸시 완료 후
  2. GitHub 웹사이트 → 저장소 페이지 이동
  3. "Pull requests" 탭 클릭
  4. "New pull request" 버튼 클릭
  5. 본인 브랜치 → develop으로 설정
Q5: 충돌이 발생했는데 무서워요! 😱
해결 방법:
  1. 침착하세요! 충돌은 자연스러운 현상입니다
  2. VS Code에서 충돌 파일을 열어보세요
  3. 색깔로 구분된 코드 중 어떤 걸 사용할지 선택
  4. 확신이 안 서면 관련 팀원과 상의하세요
  5. 해결 후 "Mark as Resolved" → 커밋/푸시
Q6: 팀원의 파일이 내 컴퓨터에 안 보여요!
해결 방법:
  1. Source Control에서 Pull 버튼 (⬇️) 클릭
  2. 또는 "..." 메뉴 → "Pull" 선택
  3. develop 브랜치로 이동해서도 Pull 해보기
  4. 그래도 안 되면 새로 Clone 받아보기

🆘 긴급 상황 대처법

뭔가 완전히 꼬였다면:
  1. 백업: 현재 작업 중인 파일들을 다른 폴더에 복사
  2. 새로 시작: 저장소를 새로 Clone 받기
  3. 파일 복원: 백업한 파일들을 새로운 폴더에 붙여넣기
  4. 다시 시작: 브랜치 생성부터 차근차근 다시
