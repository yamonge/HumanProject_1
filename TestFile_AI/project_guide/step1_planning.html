📋 1단계: 기획 단계

프로젝트의 기초를 다지는 단계

1 기획문서 작성

기획문서는 "우리가 무엇을 만들 것인가?"를 정리한 문서입니다. 마치 여행을 떠나기 전에 여행 계획서를 만드는 것과 같습니다.

📝 기획문서 템플릿

✨ 예시:
프로젝트: BookReview
목적: 읽은 책에 대한 개인적인 감상을 기록하고, 다른 사람들의 리뷰도 볼 수 있는 공간을 만들기 위해
사용자: 독서를 좋아하는 학생들과 일반인들
기능: 회원가입, 책 검색, 리뷰 작성, 평점 주기, 베스트 리뷰 보기

2 ERD (데이터베이스 설계도) 작성

ERD는 "우리가 어떤 정보들을 저장할 것인가?"를 그림으로 그린 것입니다. 마치 집을 짓기 전에 설계도를 그리는 것과 같습니다.

🔍 초보자를 위한 설명:
Entity (엔티티): 저장할 대상 → 예: 사용자, 책, 리뷰
Attribute (속성): 각 대상의 특징 → 예: 사용자의 이름, 이메일
Relationship (관계): 대상들 사이의 연결 → 예: 사용자가 리뷰를 작성한다

📊 BookReview ERD 예시

테이블명 저장할 정보 예시
Users
(사용자)		 아이디, 비밀번호, 이름, 이메일, 가입일 hong123, ******, 홍길동, hong@email.com
Books
(책)		 책ID, 제목, 저자, 출판사, 출판일, 장르 B001, 해리포터, J.K.롤링, 문학수첩, 2000, 판타지
Reviews
(리뷰)		 리뷰ID, 사용자ID, 책ID, 평점, 내용, 작성일 R001, hong123, B001, 5점, "정말 재미있어요!", 2025-08-21

🔗 관계 설명

  • 📝 한 명의 사용자는 여러 개의 리뷰를 쓸 수 있다
  • 📚 한 권의 책은 여러 개의 리뷰를 가질 수 있다
  • ⭐ 하나의 리뷰는 한 명의 사용자가 한 권의 책에 대해 쓴다

3 요구사항 정의서 작성

요구사항 정의서는 "사이트에서 할 수 있는 모든 기능들"을 목록으로 만든 것입니다. 마치 식당 메뉴판을 만드는 것과 같습니다.

📋 기능 목록 예시

👤 사용자 관리 기능

  • 회원가입 (이름, 이메일, 비밀번호 입력)
  • 로그인 (이메일, 비밀번호로 로그인)
  • 로그아웃
  • 내 정보 수정

📚 책 관리 기능

  • 책 등록 (제목, 저자, 출판사 등 정보 입력)
  • 책 목록 보기
  • 책 검색 (제목이나 저자로 검색)
  • 책 정보 수정
  • 책 삭제

⭐ 리뷰 관리 기능

  • 리뷰 작성 (평점, 감상 내용 입력)
  • 리뷰 목록 보기
  • 내가 쓴 리뷰 보기
  • 리뷰 수정
  • 리뷰 삭제

4 WBS (할 일 목록) 작성

WBS는 큰 프로젝트를 작은 할 일들로 나누어서 정리한 것입니다. 마치 방 청소를 할 때 "옷장 정리 → 책상 정리 → 바닥 청소"로 나누는 것과 같습니다.

📅 작업 계획 예시

작업 담당자 예상 시간 마감일
로그인 페이지 HTML 김철수 4시간 2025-08-25
메인 페이지 디자인 이영희 6시간 2025-08-26
회원가입 기능 박민수 8시간 2025-08-27
리뷰 작성 기능 최지혜 10시간 2025-08-28

5 기획발표 PPT 준비

지금까지 만든 계획들을 다른 사람들에게 설명하기 위한 발표자료를 만듭니다.

🎤 발표 구성 예시

  1. 프로젝트 소개 (1분) - 무엇을 만들 것인가?
  2. 팀 소개 (1분) - 누가 만들 것인가?
  3. 주요 기능 (2분) - 어떤 기능들이 있는가?
  4. 기술 스택 (1분) - 어떤 기술을 사용하는가?
  5. 개발 계획 (2분) - 언제까지 무엇을 할 것인가?
  6. 질의응답 (3분) - 궁금한 것들 답변
