📚 BookReview 프로젝트

나만의 북 리뷰 사이트 만들기
프로젝트 기간: 4주 | 기술 스택: HTML, CSS, JavaScript, jQuery | 팀 구성: 3-4명
🔥 중요! 이 가이드는 프로젝트 관리를 처음 해보는 완전 초보자를 위한 것입니다. 각 용어와 과정을 차근차근 따라가면 성공적으로 프로젝트를 완료할 수 있습니다.

1 기획 단계 (1주차)

프로젝트를 시작하기 전에 "무엇을 만들 것인가"를 명확히 정하는 단계입니다.

📅 기간: 1주일 (월요일~일요일)
👥 참여자: 팀 전체
🎯 목표: 프로젝트 전체 계획 세우기

📋 기획문서

프로젝트가 뭔지, 왜 만드는지, 누가 사용할지를 정리한 문서

📊 ERD

데이터베이스 설계도 - 어떤 정보들을 저장할지 그림으로 그리는 것

📝 요구사항 정의서

사이트에서 할 수 있는 모든 기능들을 목록으로 만든 것

🗂️ WBS

큰 일을 작은 일들로 나누어서 정리한 것 (할 일 목록)

💡 용어 설명
ERD: Entity Relationship Diagram - 데이터들 사이의 관계를 그림으로 그린 것
WBS: Work Breakdown Structure - 큰 프로젝트를 작은 작업들로 나눈 구조

✅ 1주차 체크리스트

  • 팀원 역할 분담 완료
  • 프로젝트 주제 최종 결정
  • 기획문서 1차 완성
  • ERD 초안 작성
  • 기능 목록 정리
  • 기획발표 PPT 준비
  • 기획발표 실시

2 개발 단계 (2-3주차)

실제로 웹사이트를 코딩하는 단계입니다.

📅 기간: 2주일 (2주차~3주차)
👥 참여자: 팀 전체 (역할별로 나누어서)
🎯 목표: 작동하는 웹사이트 완성

🏗️ HTML 구조

웹페이지의 뼈대 - 제목, 버튼, 입력창 등의 구조

🎨 CSS 디자인

웹페이지를 예쁘게 꾸미는 코드 - 색깔, 크기, 배치 등

⚡ JavaScript 기능

웹페이지가 동작하게 하는 코드 - 버튼 클릭, 데이터 저장 등

📱 반응형 디자인

컴퓨터, 태블릿, 휴대폰에서 모두 잘 보이게 만드는 것

✅ 2-3주차 체크리스트

  • 메인 페이지 HTML 완성
  • 전체 페이지 CSS 스타일링
  • 회원가입/로그인 기능
  • 책 등록/수정/삭제 기능
  • 리뷰 작성/수정/삭제 기능
  • 검색 기능
  • 반응형 디자인 적용
  • 브라우저 테스트

3 마무리 단계 (4주차)

완성된 프로젝트를 정리하고 발표하는 단계입니다.

📅 기간: 1주일 (4주차)
👥 참여자: 팀 전체
🎯 목표: 프로젝트 완성 및 발표

📁 최종 소스

완성된 모든 코드 파일들을 정리한 것

📖 최종문서

프로젝트 전체를 설명하는 완성본 문서

🎬 최종발표 PPT

다른 사람들에게 프로젝트를 소개할 발표자료

🎤 최종발표

완성된 프로젝트를 사람들 앞에서 발표하기

✅ 4주차 체크리스트

  • 모든 버그 수정
  • 코드 정리 및 주석 추가
  • 사용법 설명서 작성
  • 최종발표 PPT 완성
  • 발표 연습
  • 최종 발표 실시
  • 프로젝트 회고
